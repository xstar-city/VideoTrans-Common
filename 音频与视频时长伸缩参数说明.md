# 音频与视频时长伸缩参数说明

## 概述

视频翻译管线中，翻译后的 TTS 合成音频时长通常与原始音频不同。系统通过**三层分工**来吸收这种时长差异：

1. **翻译层**（LLM）：控制翻译文本的长短，决定 TTS 合成时长
2. **音频后处理层**：实际拉伸/压缩音频信号（保证音质的前提下）
3. **视频同步层**：通过 setpts 调整视频播放速度，吸收剩余差异

四个参数控制各层的伸缩范围：

| 参数 | CLI 选项 | 默认值 | 含义 |
|------|----------|--------|------|
| `max_audio_slowdown_pct` | `--max-audio-slowdown-pct` | 0.2 | 音频信号最大放慢比例（放慢至 120%） |
| `max_audio_speedup_pct` | `--max-audio-speedup-pct` | 0.3 | 音频信号最大加快比例（加快至 70%） |
| `max_video_slowdown_pct` | `--max-video-slowdown-pct` | 0.1 | 视频最大减速比例（相对原始音频时长） |
| `max_video_speedup_pct` | `--max-video-speedup-pct` | 0.2 | 视频最大加速比例（相对原始音频时长） |

## 参数分工

```
翻译允许的总范围 = 音频可调范围 + 视频可调范围
```

| 阶段 | 使用的参数 | 含义 | 计算方式 |
|------|-----------|------|---------|
| TTS 感知翻译（判断时长是否可接受） | audio + video 合计 | 给 LLM 的翻译自由度 | `[orig×(1-audio_speedup-video_speedup), orig×(1+audio_slowdown+video_slowdown)]` |
| 音频后处理（实际拉伸/压缩） | 仅 audio | 音频信号级调整（保证音质） | 将 TTS 合成音频往原始时长方向调整，最多调整 audio 范围 |
| 视频同步（setpts 伸缩） | 剩余差异（video 范围内） | 吸收音频调整后的剩余差异 | 自动按比例调整视频播放速度 |

## 具体示例

假设原始音频片段时长 **1.0s**，使用默认参数（`max_audio_slowdown=0.2`, `max_audio_speedup=0.3`, `max_video_slowdown=0.1`, `max_video_speedup=0.2`）：

### 场景：TTS 合成了 1.4s（比原音频长 40%）

```
翻译允许范围: [1.0×(1-0.3-0.2), 1.0×(1+0.2+0.1)] = [0.5s, 1.3s]
→ 1.4s 超出范围，LLM 会被要求缩短翻译文本
```

### 场景：TTS 合成了 1.2s（比原音频长 20%）

```
翻译允许范围: [0.5s, 1.3s]  →  1.2s 在范围内 ✓

音频后处理: 1.2s 加速 16.7% → 1.0s（在 audio_speedup=0.3 范围内）
→ 最终音频时长 = 1.0s，与原始一致，视频无需调整
```

### 场景：TTS 合成了 1.3s（比原音频长 30%，正好在翻译自由度上限）

```
翻译允许范围: [0.5s, 1.3s]  →  1.3s 刚好在范围内 ✓

音频后处理: 1.3s 加速 20% → 1.04s（audio_speedup 范围内：1.3×(1-0.3)=0.91s 为下限）
→ 最终音频时长 = 1.04s
→ 剩余差异 = 1.04 - 1.0 = 0.04s → 视频减速 4%（远在 video_slowdown=0.1 以内）
```

### 场景：TTS 合成了 0.6s（比原音频短 40%）

```
翻译允许范围: [0.5s, 1.3s]  →  0.6s 在范围内 ✓

音频后处理: 0.6s 放慢 20% → 0.72s（audio_slowdown 范围内）
→ 最终音频时长 = 0.72s
→ 剩余差异 = 1.0 - 0.72 = 0.28s → 视频加速 28%
  ⚠️ 超出 video_speedup=0.2（20%），但这属于翻译失败兜底
```

## 边界情况

当 TTS 感知翻译无法在总范围内调整（LLM 多次重试后仍超出），视频同步仍会**无限制兜底**——直接用 setpts 按实际音频时长调整视频。这意味着 `max_video_*` 参数只约束**翻译阶段的判断**，不限制视频同步脚本的实际行为。后续如需在视频端也做强制 clamp，可增加相关逻辑。
